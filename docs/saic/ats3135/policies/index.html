attendance & online course engagement

This class meets for 3 hours a week, but is the same number of credits as 6-hr studios. For this reason attendance is required for your passing the class. You can plan to miss class twice for no penalty. 2 missing sessions will result in no credit received — exceptions include excused absences and accommodations situations.

This is an online class and so you will obviously be on your computer during class. Your engagement is expected to be active, e.g. you’ll probably want to look things up and code along during class, or code along with a demo. Checking email, social media, or doing work for other courses is not active engagement.

I do not require that you turn your camera on for the course, but please keep in mind that such modes of presencing and contribution can promote a more fun and engaging atmosphere for the entire group. I do require that you be prepared to answer questions during class, either over the microphone in chat.

plagiarism and academic dishonesty

As in all your classes, you’re expected to do your own work for assignments, projects, and presentations. When borrowing or referring to the ideas of others, cite the author of that work (even if informally).

In coding culture, we often learn to code by imitating the work of others in tutorials, walkthroughs, and examples. This mode of learning is encouraged and is a crucial part of internalizing the rhythm, cadence, and style of writing coding. However there are three important things you must always keep in mind:

  1. 📑**CITE BORROWED CODE** // It is common practice to use other people’s code to bolster or bootstrap your own. For instance, p5.js is a library of code written by a community of people that we will use to skip straight to a certain level of expression. In commercial and artistic programming, it is common practice to use blocks of code that do generic things (such as detect if your mouse cursor is over a shape). If your work is based off or uses portions someone else’s code, you should cite that in your code. Leave a comment at the top of your code to point to where you learned these techniques. This constitutes being honest about your work.
  2. ❔**ASK YOURSELF: DOES IT MAKE SENSE TO BORROW CODE?** // Remember that this is a class where your technical skill and aesthetic creativity are being evaluated. To that end, if you borrow a block of code from someone, not only should As the class proceeds, you are expected to develop a facility with these techniques such that you can combine them in expressive ways and step away from “bare bones” tutorials. If your work does not meaningfully diverge from the code you are based on, then your technical skills and creativity are not being showcased and they are not possible to assess.
  3. 🚨 **BS DETECTOR** // It is extremely obvious to the instructor when you attempt to pass off code as your own. The consequences for this are an immediate failing of the assignment. In egregious or repeated cases, it will trigger your failure in the course.

    If you need clarity about whether some code is OK to use, just ask. You may also consult SAIC’s academic integrity policy.
